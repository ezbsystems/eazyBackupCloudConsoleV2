e3 Cloud Backup — UI Component Guide

Job Card Redesign

Backup Jobs Page — Card Component Specification

A redesigned job card that solves visual clutter, reduces badge overuse, improves information hierarchy, and handles responsive layouts gracefully from mobile to wide desktop.

Section 01 — Analysis

Problems Identified

An audit of the current job cards across the three breakpoints provided reveals several compounding issues that hurt scannability and readability.

Current Issues
Badge overload. Active, Sync, and source type (aws) all use coloured badges with different hues — green, blue, purple — creating visual noise where the eye can't prioritise.
Flat metadata layout. Six data fields sit in a single flat row with no visual separators. Labels and values blend together, especially at narrower widths.
Redundant "Active" badge. Every active job shows an Active badge, which adds no information — the default state shouldn't need a label.
Breakpoint squeeze. Between 1281px–1450px, text wraps and the card becomes cramped. The responsive jump to vertical layout at ≤1280px is good, but the in-between zone suffers.
Button density. Run Now plus five icon buttons crowd the header area, competing with the job name for attention.
Design Principles Applied
No badge = active. Remove the Active badge entirely. Only show a "Paused" state when a job is paused, using amber to signal attention needed.
Text over badges. Mode (Sync/Archive) and Source type are plain text — they're metadata, not status indicators. No colour needed.
Colour only for status. The only colour in the card body is the last-run status dot: green for success, red for failed, amber for warning, blue pulse for running.
Two-zone layout. A header row (name + actions) and a metadata grid (6 columns with subtle dividers) creates clear visual hierarchy.
Graceful responsive. The 6-col grid flows to 3×2 at ≤1280px and 2×3 at ≤768px using CSS grid, eliminating the text-wrapping squeeze zone.
Section 02 — Colour Rules

When Colour Is Allowed

The redesigned card uses colour in exactly two places: the primary CTA button and the last-run status indicator. Everything else is expressed through the neutral text hierarchy (primary → secondary → muted).

Success — last run completed
Failed — last run errored
Warning — completed with warnings
Running — job currently executing
Paused — amber badge + highlighted button
Run Now — primary CTA only
Why remove the mode badges? "Sync" and "Archive" are not states that need attention — they're fixed properties of the job that don't change. Using a coloured badge for them elevates them to the same visual importance as actual status changes like "Failed" or "Running", which dilutes the signal when something actually needs attention.

Responsive Breakpoints

BreakpointBody GridBehaviour
> 1280px6 columnsFull horizontal layout with column dividers
769px – 1280px3 columns × 2 rowsColumns 1–3 on first row, 4–6 on second. Dividers adapt.
≤ 768px2 columns × 3 rowsHeader wraps, actions move to second row below name.
Key change for the 1281–1450px squeeze zone: Instead of keeping 6 columns and letting text wrap, the grid now breaks to 3 columns at 1280px. This eliminates the awkward mid-range where destination paths and timestamps were getting truncated and wrapping. The trade-off — an extra row of metadata — is far better than unreadable text.
Section 05 — Summary

Badge Reduction Summary

A side-by-side comparison of what was removed and what replaces it.

Old ElementOld TreatmentNew TreatmentWhy
"Active" badge Green pill with dot Removed entirely Default state needs no indicator. Absence of "Paused" = active.
"Sync" / "Archive" mode Blue badge Plain text in metadata grid Fixed property, not a status. Doesn't need colour emphasis.
Source type "(aws)" Coloured sub-badge Muted sub-text under source value Supplementary info, not a primary data point.
"Paused" state Active badge turns from green to grey Amber badge + amber pause button highlight Paused is exceptional — it deserves attention. Amber signals "needs action".
Last run "Success" Green badge Small green dot + text Reduces visual weight while maintaining the status signal.
